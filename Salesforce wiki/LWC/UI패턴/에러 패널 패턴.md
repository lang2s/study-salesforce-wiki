# 에러 패널 패턴

> `errorPanel` 컴포넌트 + `ldsUtils.reduceErrors()`로 다양한 에러 형식을 통일된 UI로 표시.

---

## errorPanel 컴포넌트 사용법

```html
<!-- 기본: SVG 일러스트레이션 + 에러 메시지 -->
<c-error-panel errors={error}></c-error-panel>

<!-- 인라인 메시지 (경량) -->
<c-error-panel errors={error} type="inlineMessage"></c-error-panel>

<!-- 친화적 메시지 커스터마이징 -->
<c-error-panel
    errors={error}
    friendly-message="데이터를 불러올 수 없습니다"
    type="inlineMessage"
>
</c-error-panel>
```

---

## errorPanel 내부 구현

```javascript
// errorPanel.js
import { reduceErrors } from 'c/ldsUtils';

export default class ErrorPanel extends LightningElement {
    @api errors;
    @api friendlyMessage = 'Error retrieving data';
    @api type; // 'inlineMessage' | undefined

    // reduceErrors로 모든 에러 타입 → string[]
    get errorMessages() {
        return reduceErrors(this.errors);
    }

    // 동적 템플릿 (type에 따라 렌더링 방식 변경)
    render() {
        if (this.type === 'inlineMessage') return inlineMessage;
        return noDataIllustration; // 기본: SVG 일러스트레이션
    }
}
```

## 에러 상태 관리 패턴

```javascript
// wire 에러
@wire(getContactList)
wiredContacts({ data, error }) {
    if (data) {
        this.contacts = data;
        this.error = undefined;
    } else if (error) {
        this.error = error;
        this.contacts = undefined;
    }
}

// imperative 에러
async handleLoad() {
    try {
        this.data = await loadData();
        this.error = undefined;
    } catch (error) {
        this.error = error;
        this.data = undefined;
    }
}
```

```html
<template lwc:if={data}>
    <!-- 정상 표시 -->
</template>
<template lwc:elseif={error}>
    <c-error-panel errors={error}></c-error-panel>
</template>
<template lwc:else>
    <lightning-spinner></lightning-spinner>
</template>
```

## 에러 타입별 추출 구조

```javascript
// reduceErrors가 처리하는 8가지 에러 타입
// 1. UI API read error: error.body[] (Array)
// 2. 페이지 레벨: error.body.pageErrors[].message
// 3. 필드 레벨: error.body.fieldErrors[field][].message
// 4. DML 페이지: error.body.output.errors[].message
// 5. DML 필드: error.body.output.fieldErrors[field][].message
// 6. Apex/네트워크: error.body.message
// 7. JS Error: error.message
// 8. HTTP: error.statusText
```
