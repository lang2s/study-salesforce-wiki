# 상태 관리

> 단일 컴포넌트 내 상태는 reactive property(setter), 여러 컴포넌트 공유는 `@lwc/state`, DOM 경계 초월은 LMS.

---

## 전략 선택

| 범위 | 방법 |
|---|---|
| 단일 컴포넌트 내 | reactive property + setter |
| 부모-자식 간 | @api property + CustomEvent |
| 형제/크로스 컴포넌트 | `@lwc/state` (최신) 또는 [[Lightning Message Service]] |
| Aura/VF 포함 | [[Lightning Message Service]] |

## 패턴 1: Setter 기반 Reactive (단일 컴포넌트)

```javascript
export default class TodoList extends LightningElement {
    _todos = [];
    filteredTodos = [];

    @api
    get todos() { return this._todos; }
    set todos(value) {
        this._todos = value;
        this.filterTodos(); // 값 변경 시 자동 파생 상태 갱신
    }

    filterTodos() {
        this.filteredTodos = this.priorityFilter
            ? this._todos.filter(t => t.priority)
            : this._todos;
    }
}
```

## 패턴 2: @lwc/state (여러 컴포넌트 공유 상태)

### State 정의 (stateManager 컴포넌트)

```javascript
// opportunitiesStateManager.js
import { defineState } from '@lwc/state';

export default defineState(({ atom, computed, setAtom }) => {
    // atom: 원자적 상태 (단일 진실 소스)
    const opportunities = atom([]);

    // setter 함수
    const setOpportunities = (newOpps) => {
        setAtom(opportunities, newOpps);
    };

    // computed: 자동 메모이제이션된 파생 상태
    const totalAmount = computed(
        [opportunities],
        (opps) => opps.reduce((sum, o) => sum + o.Amount.value, 0)
    );

    return { opportunities, setOpportunities, totalAmount };
});
```

### State 사용 (소비 컴포넌트)

```javascript
// opportunitiesList.js
import { fromContext } from '@lwc/state';
import opportunitiesStateManager from 'c/opportunitiesStateManager';

export default class OpportunitiesList extends LightningElement {
    stateManager = fromContext(opportunitiesStateManager);

    get opportunities() {
        // stateManager.value → 현재 상태 접근
        return this.stateManager.value.opportunities.map(o => ({
            id: o.Id,
            name: o.Name.value,
            amount: o.Amount.value
        }));
    }
}

// opportunitiesSummary.js (다른 컴포넌트)
export default class OpportunitiesSummary extends LightningElement {
    stateManager = fromContext(opportunitiesStateManager);

    get totalAmount() {
        return this.stateManager.value.totalAmount; // computed 자동 메모이제이션
    }
}
```

### State 업데이트

```javascript
// stateManager.js (데이터 로드 및 업데이트)
export default class StateManager extends LightningElement {
    state = opportunitiesStateManager();

    @wire(graphql, { query: gql`...`, variables: '$queryVariables' })
    wiredOpportunities({ data, errors }) {
        if (data) {
            this.state.value.setOpportunities(
                data.uiapi.query.Opportunity.edges.map(e => e.node)
            );
        }
    }
}
```

## @lwc/state 핵심 개념

| 개념 | 설명 |
|---|---|
| `atom(initialValue)` | 원자적 상태 — 단일 진실 소스 |
| `computed([deps], fn)` | 의존 atom 변경 시 자동 재계산, 메모이제이션 |
| `setAtom(atom, value)` | atom 값 변경 |
| `fromContext(manager)` | 소비 컴포넌트에서 state 접근 |
| `stateManager.value` | 현재 상태 스냅샷 |

## Reactive Property 변경 감지 규칙

```javascript
// ✅ LWC가 감지하는 변경 (새 참조)
this.name = 'newValue';          // primitive
this.items = [...this.items];    // 새 배열
this.config = { ...this.config, key: val }; // 새 객체

// ❌ LWC가 감지 못하는 변경 (같은 참조)
this.items.push(newItem);        // 배열 직접 수정
this.config.key = val;           // 객체 속성 직접 수정
```
