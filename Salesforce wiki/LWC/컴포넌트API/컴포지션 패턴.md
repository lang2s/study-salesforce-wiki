# 컴포지션 패턴

> LWC 컴포넌트는 Container(데이터/상태 소유)와 Presentational(렌더링만) 역할로 분리. 재사용성과 테스트 용이성을 높인다.

---

## Container vs Presentational

| | Container | Presentational |
|---|---|---|
| 역할 | 데이터 로드, 상태 관리 | 전달받은 데이터 표시 |
| Apex 호출 | ✅ (wire/imperative) | ❌ |
| @api | 적음 | 많음 |
| 예시 | `compositionContactSearch` | `contactTile`, `contactListItem` |

---

## 단순 단일 자식 (Basics)

```javascript
// compositionBasics.js — Container
export default class CompositionBasics extends LightningElement {
    contact = {
        Name: 'Amy Taylor',
        Title: 'VP of Engineering',
        Phone: '6172559632'
    };
}
```

```html
<!-- Container가 데이터 소유, Presentational에 전달 -->
<c-contact-tile contact={contact}></c-contact-tile>
```

---

## 배열 반복 (for:each)

```html
<!-- key 필수 — 가상 DOM 최적화 -->
<template for:each={contacts} for:item="contact">
    <c-contact-tile key={contact.Id} contact={contact}></c-contact-tile>
</template>
```

> [!warning] key 없으면 경고
> `for:each` 루프의 루트 요소에는 반드시 `key={unique_id}` 필요. Id 필드 권장, 없으면 index 사용 (성능 저하 감수).

## Smart Container (Apex 데이터 로드)

```javascript
// compositionContactSearch.js — 검색 + 로드 + 에러 처리
export default class CompositionContactSearch extends LightningElement {
    contacts;
    error;

    handleKeyChange(event) {
        window.clearTimeout(this.delayTimeout);
        const searchKey = event.target.value;
        this.delayTimeout = setTimeout(async () => {
            try {
                this.contacts = await findContacts({ searchKey });
                this.error = undefined;
            } catch (error) {
                this.error = error;
                this.contacts = undefined;
            }
        }, 300);
    }
}
```

```html
<lightning-input type="search" onchange={handleKeyChange} label="Search"></lightning-input>

<template lwc:if={contacts}>
    <template for:each={contacts} for:item="contact">
        <c-contact-tile key={contact.Id} contact={contact}></c-contact-tile>
    </template>
</template>
<template lwc:elseif={error}>
    <c-error-panel errors={error}></c-error-panel>
</template>
```

## 조건부 렌더링 디렉티브

```html
<!-- lwc:if + lwc:elseif + lwc:else -->
<template lwc:if={isLoading}>
    <lightning-spinner></lightning-spinner>
</template>
<template lwc:elseif={contacts}>
    <!-- 데이터 표시 -->
</template>
<template lwc:else>
    <p>검색 결과 없음</p>
</template>
```

> [!note] `if:true` vs `lwc:if`
> 구형 API: `if:true={condition}` → 신형 API: `lwc:if={condition}`. API 59.0(Winter '23)부터 `lwc:if` 권장. `lwc:elseif`, `lwc:else` 지원.

## Immutable Data 패턴

```javascript
// ❌ 배열 직접 mutation → @api setter 트리거 안 됨
this.todos.push(newTodo);

// ✅ 새 배열 생성 → 반응성 트리거
this.todos = [...this.todos, newTodo];

// ✅ 객체도 마찬가지
this.contact = { ...this.contact, Name: newName };
```
